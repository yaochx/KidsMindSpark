# Codex 开发硬约束合同

本文件是后续 Codex 开发的最高优先级约束。每次开发前必须先阅读本文件，再阅读当前 milestone 文档。

## 不可违反的产品约束

1. 故事优先页数预算
   - 漫画正文不再固定为 32 页，必须使用 M14 的 `story_first_bounded` 策略。
   - 单个故事页数必须在 16-48 页之间，由故事表达需要决定。
   - 单个故事最多 96 个分镜，每页必须 1-4 个分镜。
   - 旧的 32 页故事仍然合法，但 32 页不再是硬编码目标。

2. 主线确认先于正文生成
   - 用户必须先看到图形化故事主线确认页。
   - 用户确认或保存主线后，才允许生成漫画分镜。
   - 不允许从故事输入直接跳到正文、预览或 PDF。

3. 不允许无限续写
   - 不提供“继续写”“无限扩展”“再来一章”等无边界生成入口。
   - 所有生成动作必须落在固定流程和固定数据模型内。

4. 不允许纯文本 PDF 作为最终目标
   - PDF 必须服务于“日式/中式彩色漫画”输出。
   - MVP 可使用 mock 图片或色块占位。
   - PDF 至少应包含页码、分镜、图像占位、角色对白、旁白等漫画结构。

5. UI 不允许重叠
   - 正文、卡片、节点、按钮、导航、状态提示不得互相覆盖。
   - 所有页面必须在桌面和常见移动宽度下检查布局。
   - Mermaid/React Flow 节点必须有明确尺寸、间距和滚动策略。

6. 所有页面必须组件化
   - 页面级文件只做编排。
   - 输入表单、主线节点、分镜页卡、漫画预览页、导出控件等必须拆成组件。
   - 不允许把完整页面 UI 和业务逻辑堆在一个超大文件里。

7. 先 mock，后真实 API
   - MVP 优先使用 mock provider。
   - API、数据模型和 provider interface 必须为后续真实 AI 与图像生成预留替换点。
   - 不在早期 milestone 中强行接入真实外部服务。

8. 每次开发按 M0-M6 小步提交
   - 每次只实现一个 milestone。
   - 不要一次性生成整个项目。
   - 每个 milestone 完成后运行可用测试或静态检查，并更新 README 或对应文档。

9. ImageProvider prompt 构建边界
   - ImageProvider 可以基于 `story/page/panel` 构建图片 prompt。
   - ImageProvider 和 prompt builder 不得修改故事、页数、分镜、对白或主线确认状态。
   - `panel.imagePrompt` 是核心画面描述，但不能作为真实图片 provider 的唯一 prompt 内容长期使用。

10. M11 Panel Prompt Builder 共用约束
   - M11 必须提供统一 `build_panel_image_prompt(story, page, panel)`。
   - M11 必须为构建后的 prompt 生成稳定 `promptHash`，供 M12/M13 缓存和候选图管理使用。
   - OpenAI ImageProvider 和豆包 Seedream ImageProvider 必须共用该 prompt builder。
   - prompt 必须服务于单格漫画画面，不允许生成整页拼图或整本故事。
   - 默认策略是图片模型生成单格画面和中文漫画对白气泡，应用负责拼页、边框、页码和 PDF。
   - 对白气泡内只允许写 `dialogue.text`，不得写角色名、冒号、编号或额外旁白。
   - 说话角色必须通过气泡尾巴、指向线和靠近对应角色的位置表达。

11. M12 Image Asset Cache 与导出约束
   - 真实生图产物必须保存为可复用 Image Asset，不得只作为一次性临时结果。
   - 同一 panel 可以保留多张候选图，必须有明确 `selectedImageId`。
   - 缓存命中时不得重复调用真实文生图 API。
   - 已选图片必须按 `panelId` 显示在对应分镜框内。
   - PDF 嵌图不得改变故事、页数、分镜、对白或主线确认状态。
   - 图片读取接口不得暴露任意本地文件访问。
   - 图片缺失或读取失败时必须保留占位，不得中断整本导出。

12. M13 批量生成与一键自动化约束
   - 一键自动化必须受预算限制，不允许无限制消耗 token 或图片额度。
   - 批量任务必须先查缓存，再为缓存未命中的 panel 调用真实生图。
   - 批量任务必须支持按 panel 查看状态、失败重试、候选图挑选和单格结果替换。
   - 用户可以调整单格 prompt 并重生成该 panel，但不得借此改写故事结构。
   - 批量任务不得绕过 M11 prompt builder。

13. M14 故事优先预算约束
   - 固定 32 页已在 M14 通过完整产品约束变更替换，后续不得重新写死为 32 页。
   - 必须有页数范围、最大 panel 数、单故事生图预算、单批生图预算和单 panel 候选图上限。
   - 本地 MVP 可使用 `workspaceId=local_default` 和 `projectId` 管理缓存与预算，不要求正式账号系统。

## Codex 自检清单

- 是否先阅读 `docs/AI_CONTRACT.md`？
- 是否只实现当前 milestone？
- 是否保留“主线确认先于正文生成”？
- 是否仍满足 16-48 页、最多 96 个分镜和生图预算？
- 是否避免了纯聊天式 UI？
- 是否避免了纯文本 PDF？
- 是否保持组件化？
- 是否没有引入无限续写入口？
- 是否没有写入无关功能？
- 是否没有安装未被当前 milestone 需要的依赖？
